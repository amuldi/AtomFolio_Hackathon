# AtomFolio +

투자 CSV를 업로드하면 포트폴리오를 자동으로 해석하고, 보유 종목의 구조와 성과를 하나의 인터랙티브 대시보드로 보여주는 웹 애플리케이션입니다. 사용자는 복잡한 표 데이터를 정리하지 않아도 종목, 수익률, 자산 비중, 날짜별 흐름, 포트폴리오 점수를 시각적으로 확인할 수 있습니다.

## 프로젝트 요약

AtomFolio +는 투자 데이터를 "읽기 쉬운 형태"로 바꾸는 것에 초점을 둔 대시보드입니다. CSV 안의 컬럼명이 한국어/영어로 섞여 있거나, 날짜와 수익률 형식이 일정하지 않아도 핵심 컬럼을 추론하고 포트폴리오 분석 화면을 구성합니다.

주요 목표는 다음과 같습니다.

- 투자 CSV의 종목, 날짜, 수익률, 계좌, 비중 컬럼을 자동 매핑
- 종목명과 코드 기반으로 지역, 섹터, 스타일, 위험 등급, 자산군 보강
- 보유 종목을 원자형 인터랙션 UI로 시각화
- 수익 흐름, 자산 배분, 포트폴리오 점수를 한 화면에서 확인
- 해커톤 주제인 투자 데이터 활용 및 스킬형 대시보드에 맞는 분석 경험 제공

## 핵심 기능

- **CSV 자동 인식**: `종목명`, `ticker`, `date`, `수익률`, `비중`, `계좌유형`처럼 다양한 컬럼명을 표준 필드로 추론합니다.
- **포트폴리오 원자 뷰**: 각 종목을 하나의 원자로 표현하고, 수익률/분류 정보에 따라 선택과 하이라이트가 가능합니다.
- **그룹 하이라이트**: 투자 지역, 분야, 스타일, 위험 등급 기준으로 종목을 빠르게 구분합니다.
- **수익 캘린더 히트맵**: 날짜별 수익률 또는 손익 데이터를 잔디 차트 형태로 보여줍니다.
- **자산 비중 도넛 차트**: 종목, 자산군, 계좌 기준 비중을 도넛 차트와 범례로 제공합니다.
- **스파이더 점수 차트**: 수익성, 분산투자, 위험관리, 포트폴리오 구성, 투자 타이밍, 수익 안정성을 자동 점수화합니다.
- **설정 패널**: 언어, 자산군 분류 방식, 도넛 차트 기준, 점수 가중치 프리셋을 변경할 수 있습니다.

## 구현하며 노력한 점

- **불완전한 CSV 대응**: 실제 투자 데이터는 컬럼명이 제각각이기 때문에, 정확히 같은 헤더만 받는 방식이 아니라 유사 키워드와 한국어/영어 표현을 폭넓게 인식하도록 구현했습니다.
- **데이터 보강 로직**: 종목명만 있는 데이터도 대시보드에서 의미 있게 보이도록 지역, 분야, 스타일, 위험 등급, 자산군을 추론하는 보강 레이어를 만들었습니다.
- **분석 신뢰도 점검**: 서버에는 `schema-mapper`, `quality-guard`, `explanation` 역할의 결정론적 에이전트 흐름을 두어 컬럼 매핑, 데이터 품질, 사용자 설명을 분리했습니다.
- **점수 계산의 설명 가능성**: 단순히 점수만 보여주지 않고 각 축이 어떤 데이터에서 계산됐는지 설명 문구를 생성하도록 구성했습니다.
- **여러 데이터 형태 지원**: 날짜가 없는 정적 포트폴리오, 날짜별 수익률 시계열, 비중이 있는 자산 구성 데이터 모두를 처리할 수 있게 했습니다.
- **인터랙션 안정성**: 플로팅 도구, 스파이더 차트, 도넛 차트가 화면 밖으로 밀리지 않도록 위치 보정과 재정렬 로직을 넣었습니다.

## UI / UX 설계

전체 UI는 투자 데이터를 딱딱한 표가 아니라 "살아 있는 포트폴리오 지도"처럼 탐색하도록 설계했습니다.

- **어두운 우주형 배경**: 포트폴리오를 원자와 궤도처럼 표현하기 위해 검은 배경, 손그림 느낌의 선, 은은한 빛을 사용했습니다.
- **원자형 종목 배치**: 각 종목이 독립된 점이 아니라 포트폴리오를 이루는 구성 요소처럼 느껴지도록 중심 구조와 움직임을 적용했습니다.
- **플로팅 도구 UI**: `+` 버튼 아래에 그룹, 히트맵, 스파이더, 도넛 도구가 정렬되며 필요한 분석만 펼쳐 볼 수 있습니다.
- **스파이더 차트**: 6개 투자 역량 축을 한 번에 비교할 수 있도록 만들었고, 다시 누르면 접히도록 토글 동작을 다듬었습니다.
- **도넛 차트**: 비중이 큰 자산과 작은 자산의 차이를 직관적으로 보이게 하고, 아이콘과 차트 패널이 가까이 붙도록 배치했습니다.
- **반응형 보정**: 데스크톱과 모바일에서 도구 크기, 패널 폭, 화면 가장자리 보정을 다르게 적용해 UI가 잘리지 않도록 했습니다.

```bash
npm run build
npm run preview
```

## 사용 방법

1. 화면의 업로드 버튼으로 투자 CSV 파일을 선택합니다.
2. 업로드 후 원자형 포트폴리오 뷰에서 종목 구성을 확인합니다.
3. `+` 도구 버튼을 눌러 그룹 하이라이트, 히트맵, 스파이더 차트, 도넛 차트를 확인합니다.
4. 설정 버튼에서 언어, 자산군 분류 방식, 도넛 차트 기준, 점수 가중치를 조정합니다.
5. 여러 포트폴리오는 최대 7개까지 업로드해 비교할 수 있습니다.

## 지원 데이터 예시

`samples/portfolio/` 폴더에 검증용 CSV가 포함되어 있습니다.

- `portfolio_test0.csv`: 날짜 없는 포트폴리오 구성 데이터
- `portfolio_test1.csv` ~ `portfolio_test4.csv`: 날짜별 수익률 시계열 데이터
- `portfolio_test5.csv`: 히트맵이 비는 케이스 확인용 데이터

인식 가능한 주요 컬럼 예시는 다음과 같습니다.

- 종목: `종목명`, `자산명`, `상품명`, `ticker`, `symbol`, `securityName`
- 날짜: `날짜`, `일자`, `거래일`, `매수일`, `date`, `tradeDate`
- 수익: `수익률`, `일일수익률(%)`, `누적수익률(%)`, `손익`, `return`, `pnl`
- 비중/금액: `비중`, `자산비중(%)`, `평가금액`, `매수가`, `보유수량`
- 분류: `계좌유형`, `투자지역`, `분야`, `투자스타일`, `위험등급`, `자산구분`

## 기술 스택

- React
- Vite
- Three.js / React Three Fiber
- Node.js 로컬 서버
- CSV 파싱 및 포트폴리오 분석 유틸리티

## 프로젝트 구조

```text
src/
  App.jsx                         # 메인 UI, 플로팅 도구, 인터랙션
  styles.css                      # 전체 화면 스타일과 반응형 UI
  lib/
    portfolioIngestionCore.js     # CSV 파싱, 컬럼 추론, 표시 데이터 생성
    securityKnowledge.js          # 종목 메타데이터 보강
    portfolioHeatmap.js           # 수익 히트맵 데이터 생성
    portfolioAllocation.js        # 도넛 차트 비중 계산
    portfolioScoring.js           # 스파이더 점수 계산
server/
  index.mjs                       # 로컬 API 서버
  dev.mjs                         # 개발 서버 실행
  portfolioIngestion.mjs          # 서버 측 포트폴리오 수집 파이프라인
  agents/                         # 스키마/품질/설명 검토 에이전트
samples/portfolio/                # 샘플 투자 CSV
docs/proposal/                    # 해커톤 제출용 기획서
Skills.md                         # 재현 가능한 구현/분석 절차 문서
```
