# AtomFolio +

투자 CSV를 업로드하면 포트폴리오의 종목 구성, 수익 흐름, 자산 비중, 위험/분산 상태를 자동으로 분석하고 시각화하는 웹 대시보드입니다. 고정된 CSV 템플릿을 요구하지 않고, `Skills.md`에 정의한 분석 규칙을 기반으로 다양한 투자 데이터 구조를 해석합니다.

## 프로젝트 개요

AtomFolio +의 목표는 복잡한 투자 표를 바로 탐색 가능한 포트폴리오 화면으로 바꾸는 것입니다. 사용자는 증권사, 자산관리 앱, 직접 만든 표에서 내려받은 CSV를 업로드한 뒤 종목별 수익률, 날짜별 흐름, 자산군 비중, 포트폴리오 점수를 한 화면에서 확인할 수 있습니다.

| 항목 | 내용 |
| --- | --- |
| 프로젝트명 | AtomFolio + |
| 형태 | React + Vite 기반 단일 페이지 웹앱 |
| 보조 서버 | Node.js HTTP API |
| 핵심 문서 | `Skills.md` |
| 외부 API 키 | 필요 없음 |
| 주요 제출물 | 개발 산출물 ZIP, 기획서 PDF, `Skills.md` |

## 핵심 가치

- **CSV 구조 자동 추론**: 한국어/영어 컬럼명, 날짜 형식, 수익률 표기, 비중/금액 컬럼을 폭넓게 인식합니다.
- **원자형 포트폴리오 시각화**: 보유 종목을 하나의 원자 구조처럼 배치해 표보다 직관적으로 탐색할 수 있습니다.
- **시계열과 표시 데이터 분리**: 날짜별 반복 행은 히트맵에 유지하고, 화면에는 종목 단위로 축약해 정보 과밀을 줄입니다.
- **분석 규칙의 재현성**: 데이터 처리 기준을 `Skills.md`에 정리해 같은 방식으로 결과물을 다시 만들 수 있습니다.
- **설명 가능한 점수화**: 수익성, 분산투자, 위험관리, 포트폴리오 구성, 투자 타이밍, 수익 안정성을 축별로 계산합니다.

## 구현 내용

### 1. 투자 CSV 업로드와 자동 파싱

- CSV, TSV, 세미콜론, 파이프 구분 텍스트를 처리합니다.
- UTF-8을 우선 사용하고, 깨짐 문자가 많은 경우 EUC-KR 디코딩을 시도합니다.
- `종목명`, `ticker`, `date`, `수익률`, `손익`, `비중`, `계좌유형`처럼 다양한 컬럼명을 표준 필드로 매핑합니다.
- 헤더가 명확하지 않은 파일도 본문 값 패턴으로 날짜, 종목, 수익률 후보를 추론합니다.
- 업로드 본문은 서버에서 8MB 이하로 제한합니다.

### 2. 종목명 오인 방지와 메타데이터 보강

- `미국`, `선진국`, `기술`, `고위험`, `40` 같은 값이 종목명으로 잘못 잡히지 않도록 제외 규칙을 두었습니다.
- `TIGER`, `KODEX`, `SPDR`, `iShares`, `Vanguard`, `NASDAQ`, `S&P`, `ETF` 같은 단서는 종목명 또는 상품명 후보로 인정합니다.
- 종목명 또는 종목코드 기반으로 지역, 분야, 투자 스타일, 위험 등급, 자산군, 통화, 변동성, 비교 지수를 보강합니다.
- 보강 데이터는 종목 메타데이터에만 적용하며, 행별 날짜/수익률/손익 원본 값은 덮어쓰지 않습니다.

### 3. 원자형 포트폴리오 UI

- 중앙 원자는 포트폴리오 전체, 주변 원자는 개별 종목을 의미합니다.
- 종목 라벨은 종목명과 수익률 중심으로 간결하게 표시합니다.
- hover 카드에서 종목명, 종목코드, 계좌유형, 매수일, 매수가, 수량, 수익률, 지역, 분야, 스타일, 위험, 자산군 정보를 확인할 수 있습니다.
- 클릭한 종목과 같은 그룹의 종목을 함께 강조해 포트폴리오 편중을 빠르게 파악할 수 있습니다.

### 4. 플로팅 분석 도구

- `+` 버튼 아래에 그룹, 히트맵, 레이더 점수, 도넛 차트 도구를 배치했습니다.
- 각 도구는 펼침/접힘 상태를 갖고, 화면 가장자리에서 잘리지 않도록 위치를 보정합니다.
- 데스크톱과 모바일에서 도구 크기, 패널 폭, 여백을 다르게 계산합니다.
- 사용자가 이동한 플로팅 도구 위치는 로컬 스토리지에 저장합니다.

### 5. 수익 캘린더 히트맵

- 날짜별 수익률 또는 손익을 GitHub 잔디밭 형태의 주 단위 그리드로 표시합니다.
- 기본 범위는 최근 24주입니다.
- 같은 날짜에 여러 행이 있으면 수익률/손익 흐름을 집계해 표시합니다.
- 날짜와 손익 데이터가 없으면 빈 상태 메시지를 보여줍니다.

### 6. 자산 비중 도넛 차트

- 자동, 종목별, 자산군 기준, 계좌 기준 보기를 지원합니다.
- 비중 계산 우선순위는 명시적 비중 컬럼, `매수가 * 보유수량`, 균등 비중 순서입니다.
- 도넛 중앙에는 가중 평균 총 수익률을 표시합니다.
- segment hover 시 범례와 차트가 함께 반응합니다.

### 7. 레이더 점수 차트

포트폴리오를 6개 축으로 자동 평가합니다.

| 축 | 평가 기준 |
| --- | --- |
| 수익성 | 평균 수익률, 플러스 종목 비율, 하방 변동성 |
| 분산투자 | 종목 수, 자산군, 분야, 지역, 스타일 분산 |
| 위험관리 | 고위험 비중, 저위험 비중, 방어형 비중, 집중도 |
| 포트폴리오 구성 | 메타데이터 충실도와 자산/분야 균형 |
| 투자 타이밍 | 매수일 분산, 월별 분산, 투자 기간 |
| 수익 안정성 | 변동성, 손실 종목 비율, 방어형 비중 |

지원하는 점수 가중치 프리셋은 다음과 같습니다.

- 균형 중심
- 수익 중심
- 장기수익 중심
- 안정 중심

### 8. 서버 측 진단 에이전트 흐름

서버에는 결정론적 분석 흐름을 두어 업로드 결과를 점검합니다.

| 모듈 | 역할 |
| --- | --- |
| `schemaMapper` | 파서가 어떤 컬럼을 어떤 표준 필드로 해석했는지 검토 |
| `qualityGuard` | 행 수, 날짜/수익률 인식률, 경고 상태 점검 |
| `explanationAgent` | 업로드 진단 결과를 사용자에게 설명할 문장으로 정리 |
| `orchestrator` | 스키마, 품질, 설명 결과를 하나의 리뷰 결과로 결합 |

## 데이터 처리 흐름

```text
CSV 업로드
  -> 브라우저에서 파일 텍스트 읽기
  -> /api/portfolio/ingest POST 요청
  -> parsePortfolioTextDetailed로 구분자, 헤더, 컬럼, 행 분석
  -> securityKnowledge / securityEnrichment로 종목 메타데이터 보강
  -> timelineItems에 원본 시계열 유지
  -> collapsePortfolioItemsForDisplay로 화면용 종목 축약
  -> schemaMapper / qualityGuard / explanationAgent 진단
  -> 프런트에서 원자 뷰, 히트맵, 도넛, 레이더 점수 렌더링
```

## 실행 방법

### 1. 의존성 설치

```bash
npm install
```

### 2. 개발 서버 실행

```bash
npm run dev
```

개발 서버는 프런트엔드 Vite 서버와 Node.js API 서버를 함께 실행합니다. 터미널에 표시되는 로컬 주소를 브라우저에서 열면 됩니다.

### 3. 프로덕션 빌드

```bash
npm run build
```

### 4. 빌드 결과 미리보기

```bash
npm run preview
```

`npm run preview`는 `dist/` 빌드 산출물을 Node.js 서버에서 정적으로 제공합니다.

## 사용 방법

1. 메인 화면의 업로드 버튼으로 투자 CSV 파일을 선택합니다.
2. 업로드 후 원자형 포트폴리오 뷰에서 종목 구성을 확인합니다.
3. 종목에 마우스를 올려 상세 정보를 확인합니다.
4. 종목을 클릭해 같은 지역, 분야, 스타일, 위험 등급 그룹을 강조합니다.
5. `+` 도구 버튼에서 그룹, 히트맵, 레이더 점수, 도넛 차트를 펼쳐 봅니다.
6. 설정 버튼에서 언어, 자산군 분류 방식, 도넛 차트 기준, 점수 가중치를 조정합니다.
7. 여러 CSV를 업로드해 포트폴리오별 구성을 비교합니다.

## API

개발 서버는 Node.js `http` 모듈 기반 API를 제공합니다.

| 메서드 | 경로 | 설명 |
| --- | --- | --- |
| `GET` | `/api/health` | 서버 상태 확인 |
| `POST` | `/api/portfolio/ingest` | CSV 텍스트를 파싱하고 보강/진단 결과 반환 |
| `POST` | `/api/security/enrich` | 종목 리스트 또는 식별자 기반 메타데이터 보강 |

`/api/portfolio/ingest` 요청 예시는 다음과 같습니다.

```json
{
  "fileName": "portfolio.csv",
  "text": "종목명,날짜,수익률\nTIGER 미국S&P500,2026-01-01,1.2%"
}
```
