# AtomFolio + 기획서

## 1. 서비스 개요

AtomFolio +는 투자 CSV 파일을 업로드하면 포트폴리오 구조, 수익 흐름, 자산 비중, 위험/분산 상태를 자동으로 분석해 시각화하는 투자 데이터 대시보드이다.

일반적인 투자 데이터는 증권사, 계좌, 자산 유형, 거래/평가 기준에 따라 컬럼 구조가 달라진다. 이 프로젝트는 고정된 CSV 템플릿을 요구하지 않고, 종목명·종목코드·날짜·수익률·비중·계좌 정보를 자동 추론하여 다양한 투자 데이터를 바로 분석할 수 있게 한다.

핵심 컨셉은 “복잡한 투자 표를 직관적인 원자형 포트폴리오로 바꾸는 것”이다. 각 종목은 하나의 원자로 표현되고, 보조 패널에서 수익 히트맵, 자산 비중 도넛, 레이더 점수 차트를 함께 제공한다.

## 2. 문제 정의

투자 데이터 분석에서는 다음 문제가 반복된다.

- 데이터 출처마다 컬럼명이 다르다.
- 날짜별 수익률 데이터와 현재 보유 비중 데이터가 섞여 있다.
- 종목명, 자산군, 계좌유형, 수익률의 위치가 일정하지 않다.
- 숫자 비중이나 지역명이 종목명으로 잘못 해석될 수 있다.
- 분석 기준이 문서화되지 않으면 같은 데이터를 매번 다른 방식으로 해석하게 된다.
- 표 형태의 투자 데이터는 초보자에게 직관적이지 않다.

AtomFolio +는 이러한 문제를 해결하기 위해 `Skills.md`에 분석 규칙을 정의하고, 그 규칙에 따라 자동 대시보드를 생성하는 방식을 채택한다.

## 3. 목표 사용자

### 개인 투자자

증권사 또는 자산관리 앱에서 내려받은 보유 종목 CSV를 빠르게 확인하고, 포트폴리오가 특정 자산군이나 지역에 치우쳐 있는지 파악하고 싶은 사용자.

### 금융/데이터 분석 입문자

투자 데이터를 직접 시각화하고 싶지만, CSV 구조가 일정하지 않아 전처리에서 막히는 사용자.

### 해커톤 심사자와 운영자

제공 데이터가 없는 상황에서도 더미 데이터 또는 공개 투자 데이터를 업로드해 서비스의 범용성, 자동 분석 능력, 시각화 완성도를 바로 확인할 수 있는 사용자.

## 4. 핵심 가치 제안

| 가치 | 설명 |
| --- | --- |
| 범용성 | 고정 템플릿 없이 다양한 투자 CSV 구조를 자동 추론한다. |
| 직관성 | 종목을 원자 단위로 표현해 포트폴리오 구성을 한눈에 보여준다. |
| 자동 분석 | 날짜, 수익률, 비중, 계좌, 자산군 정보를 자동으로 찾아 차트를 생성한다. |
| 설명 가능성 | 업로드 진단과 점수 설명을 통해 어떤 기준으로 분석했는지 보여준다. |
| 재현성 | `Skills.md`에 데이터 처리, 시각화, 점수 산정 규칙을 문서화한다. |

## 5. 분석 흐름 설계

### 5.1 업로드

사용자는 CSV 파일을 업로드한다. 여러 포트폴리오는 최대 7개까지 추가할 수 있다.

지원 구분자는 다음과 같다.

- 쉼표 CSV
- 탭 TSV
- 세미콜론
- 파이프

문자 인코딩은 UTF-8을 우선 사용하고, 깨짐 문자가 많으면 EUC-KR을 시도한다.

### 5.2 컬럼 자동 추론

업로드된 행에서 헤더 후보를 찾고, 각 컬럼을 표준 필드로 매핑한다.

주요 표준 필드:

- 종목명 또는 종목코드
- 날짜
- 수익률 또는 손익
- 계좌유형 또는 계좌 ID
- 매수가와 보유수량
- 비중 또는 평가금액
- 지역, 분야, 스타일, 위험 등급, 자산군

컬럼명이 명확하지 않으면 본문 값의 패턴을 사용한다. 예를 들어 `2026-01-01` 형태가 반복되면 날짜 컬럼으로 추론하고, `TIGER 미국S&P500`, `QQQM`, `AMD`처럼 종목 단서가 있으면 종목명 또는 종목코드 후보로 본다.

### 5.3 종목명 오인 방지

투자 데이터에서 `미국`, `선진국`, `대형주`, `40` 같은 값은 종목명이 아니라 메타데이터 또는 숫자일 가능성이 높다. AtomFolio +는 이런 값을 종목명 후보에서 제외한다.

반대로 `TIGER`, `KODEX`, `SPDR`, `iShares`, `Vanguard`, `NASDAQ`, `S&P`, `ETF` 같은 단서는 종목명으로 인정한다.

### 5.4 종목 메타데이터 보강

CSV에 지역, 분야, 스타일, 위험, 자산군이 없어도 종목명 또는 종목코드를 기반으로 가능한 범위에서 보강한다.

보강 대상:

- 투자 지역
- 분야
- 투자 스타일
- 위험 등급
- 자산 구분
- 통화
- 규모 분류
- 변동성
- 과세 구분
- 비교 지수

중요하게, 보강 데이터는 종목 메타데이터에만 적용한다. 행별 날짜, 수익률, 손익, 원본 필드는 덮어쓰지 않는다.

### 5.5 표시용 데이터와 시계열 데이터 분리

같은 종목이 날짜만 다르게 반복되는 경우가 많다. 예를 들어 90일치 `TIGER 미국S&P500` 수익률이 있으면 화면에 90개의 원자를 만들지 않고, 표시용 원자는 하나로 묶는다.

대신 날짜별 원본 행은 `timelineItems`로 유지하고, 수익 캘린더 히트맵은 이 데이터를 사용한다.

## 6. 대시보드 구성

### 6.1 원자형 포트폴리오 뷰

중앙의 원자형 인터페이스는 포트폴리오의 전체 구조를 상징적으로 보여준다.

- 중앙 원자: 포트폴리오 전체
- 주변 원자: 개별 종목
- 라벨: 종목명과 수익률
- 인터랙션: hover 정보 카드, 클릭 강조, 드래그/회전

이 방식은 표 형태의 금융 데이터를 시각적으로 탐색할 수 있게 한다.

### 6.2 그룹 하이라이트

다음 기준으로 종목을 강조한다.

- 투자 지역
- 분야
- 투자 스타일
- 위험 등급

사용자가 특정 종목을 선택하면 같은 그룹에 속한 종목이 함께 강조되어 포트폴리오 편중 여부를 확인할 수 있다.

### 6.3 수익 캘린더 히트맵

날짜별 수익률 또는 손익 흐름을 GitHub 잔디밭 형태로 보여준다.

- 양수 수익: 더 진한 긍정 색상
- 음수 수익: 손실 색상
- 데이터 없음: 빈 셀
- hover: 날짜와 값 표시

시계열 데이터가 없으면 “날짜와 손익 데이터가 없어 히트맵을 표시할 수 없습니다”라는 빈 상태를 보여준다.

### 6.4 자산 비중 도넛 차트

포트폴리오 구성을 도넛 차트로 표시한다.

지원 기준:

- 자동
- 종목별 비중
- 자산군 기준
- 계좌 기준

비중 계산 우선순위:

1. 명시적 비중 컬럼
2. 매수가 × 보유수량
3. 균등 비중

도넛 중앙에는 가중 평균 총 수익률을 표시한다.

### 6.5 레이더 점수 차트

포트폴리오 품질을 6개 축으로 평가한다.

| 축 | 의미 |
| --- | --- |
| 수익성 | 평균 수익률, 플러스 종목 비율, 하방 변동성 |
| 분산투자 | 종목 수, 자산군, 분야, 지역, 스타일 분산 |
| 위험관리 | 고위험 비중, 저위험 비중, 방어형 비중, 집중도 |
| 포트폴리오 구성 | 메타데이터 충실도와 자산/분야 균형 |
| 투자 타이밍 | 매수일 분산, 월별 분산, 투자 기간 |
| 수익 안정성 | 변동성, 손실 종목 비율, 방어형 비중 |

사용자는 균형 중심, 수익 중심, 장기수익 중심, 안정 중심 프리셋을 선택할 수 있다.

## 7. Skills.md 설계 방향

본 프로젝트의 `Skills.md`는 단순 설명서가 아니라, 동일한 대시보드를 다시 만들 수 있는 분석 지침서 역할을 한다.

포함 내용:

- 생성할 서비스의 목적과 기술 스택
- 화면 구성과 주요 도구
- CSV 입력 규칙
- 핵심 컬럼 매핑 규칙
- 종목명 오인 방지 규칙
- 데이터 처리 파이프라인
- 종목 메타데이터 보강 규칙
- 히트맵 분석 규칙
- 자산군 분류 규칙
- 레이더 점수 산정 규칙
- 업로드 진단 규칙
- API 규격
- 샘플 CSV 검증 기준

이 문서를 기준으로 바이브 코딩을 수행하면, 분석 로직과 UI 결과가 일관되게 유지된다.

## 8. 바이브 코딩 활용 구조

AtomFolio +는 구현 전에 분석 기준과 생성 결과를 `Skills.md`로 정의하고, 이를 개발 지침으로 사용한다.

바이브 코딩 활용 방식:

1. `Skills.md`에 데이터 해석 규칙을 먼저 정의한다.
2. 규칙에 따라 CSV 파서, 종목 보강, 차트 생성 함수를 구현한다.
3. 샘플 CSV를 반복 업로드하며 오인식 사례를 수정한다.
4. 수정된 규칙을 다시 `Skills.md`에 반영한다.
5. UI는 랜딩 페이지가 아닌 실제 대시보드 중심으로 구성한다.

이를 통해 수동으로 특정 CSV에 맞춘 일회성 구현이 아니라, 다양한 투자 데이터를 처리할 수 있는 재사용 가능한 시스템을 목표로 한다.

## 9. 기술 구조

```text
src/App.jsx                         # 메인 UI와 인터랙션
src/lib/portfolioIngestionCore.js   # CSV 파싱과 컬럼 추론
src/lib/securityKnowledge.js        # 종목 메타데이터 보강
src/lib/portfolioHeatmap.js         # 수익 캘린더 생성
src/lib/portfolioAllocation.js      # 자산 비중 계산
src/lib/portfolioScoring.js         # 포트폴리오 점수 계산
server/index.mjs                    # API 서버
server/portfolioIngestion.mjs       # 업로드 파이프라인
server/securityEnrichment.mjs       # 서버 보강 캐시
server/agents/                      # 업로드 진단 에이전트
samples/portfolio/                  # 샘플 투자 CSV
Skills.md                           # 분석/구현 규칙 문서
```

## 11. 기대 효과

- 투자 CSV를 업로드하는 즉시 분석 결과를 확인할 수 있다.
- 고정 양식 없이 여러 데이터 구조에 대응할 수 있다.
- 투자 포트폴리오의 편중, 분산, 위험, 수익 흐름을 직관적으로 파악할 수 있다.
- 분석 규칙이 `Skills.md`로 문서화되어 재현성과 확장성이 높다.
- 금융 데이터 분석을 처음 접하는 사용자도 시각적 인터페이스로 포트폴리오를 탐색할 수 있다.

## 12. 차별점

### 원자형 시각화

일반적인 표와 카드 기반 대시보드 대신 원자형 인터페이스로 포트폴리오의 연결성과 구성감을 표현한다.

### 자동 구조 추론

정해진 템플릿에 맞춘 CSV만 처리하는 것이 아니라, 컬럼명과 값 패턴을 함께 분석해 종목, 날짜, 수익률, 계좌 정보를 추론한다.

### 표시 데이터와 시계열 데이터 분리

날짜별 반복 행은 히트맵에 유지하고, 화면 원자는 종목 단위로 축약해 정보 과밀을 줄인다.

### 진단 에이전트

업로드 후 스키마 매핑과 품질 상태를 진단해, 데이터가 어떤 수준으로 분석 가능한지 알려준다.

## 13. 확장 기능 아이디어

### 벤치마크 비교

S&P500, KOSPI, NASDAQ100 등 벤치마크와 포트폴리오 수익률을 비교한다.

### 리밸런싱 제안

자산군, 지역, 위험 등급이 특정 기준보다 편중되면 리밸런싱 후보를 제안한다.

### 고급 위험 지표

최대 낙폭, 변동성, 샤프 비율, 승률, 월별 수익률 분포를 추가한다.

### PDF 리포트 출력

현재 대시보드 상태를 요약한 투자 리포트를 PDF로 저장한다.

### 다중 포트폴리오 비교

최대 7개 포트폴리오를 업로드할 수 있는 구조를 활용해 계좌별, 전략별, 기간별 비교 기능을 강화한다.

## 15. 결론

AtomFolio +는 투자 데이터 분석 규칙을 `Skills.md`로 정의하고, 그 규칙을 기반으로 투자 CSV를 자동 해석해 대시보드로 시각화하는 서비스이다.

이 프로젝트의 핵심은 단순히 차트를 그리는 것이 아니라, 다양한 투자 데이터 구조를 일관된 분석 체계로 변환하는 데 있다. 이를 통해 해커톤 주제인 “투자 데이터를 시각화하라 - Skills 기반 대시보드 설계”에 맞춰 범용성, 자동 생성, 문서 기반 재현성, 실제 활용성을 함께 보여준다.
